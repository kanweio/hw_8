

Используя generic views штуки можно использовать ООП во вьюшках, наследоваться от них, и определять один и тот же context
во всех вьюшках


1.View это  главные составляющие веб-приложений на основе Django, вьюшки обязательная часть каждого приложения,
без них приложение не сможет отвечать на запросы.

2.ListView Представление используется для отображения значений из источника данных.

3.CreateView Представление, которое отображает форму для создания объекта, повторного отображения формы с ошибками
 валидации и сохранения объекта.

4.DetailView Отвечает за просмотр отедльного объекта

5.UpdateView Форма для обновления данных

6.TemplateView Заполняет HTML файлы информацией при помощи маркеров которые мы указали в шаблоне
